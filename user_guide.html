Mental Health Support Hub - User Guide

1. Introduction

This guide provides step-by-step instructions on how to set up, run, and use the Mental Health Support Hub project in Visual Studio Code (VS Code). The project is a web-based application built with Node.js, Express.js, and Socket.io for real-time communication.

2. Prerequisites

Before running the project, ensure you have the following tools installed:

  • Node.js: Version 14 or higher. Download from https://nodejs.org/
  • Visual Studio Code: Download from https://code.visualstudio.com/
  • Git: For version control. Download from https://git-scm.com/
  • Chrome Browser: For testing the application

3. Project Setup

  1. Open VS Code.
  2. Click on "File" > "Open Folder" and select the "mental-health-hub" directory.
  3. Open the integrated terminal in VS Code by pressing Ctrl + ` (or View > Terminal).
  4. Navigate to the project directory if not already there:
    cd mental-health-hub
  5. Install the project dependencies:
    npm install

4. Running the Project

  1. In the VS Code terminal, start the server:
    npm start
  2. The server will start and display: "Mental Health Hub server running on port 3001"
  3. Open your web browser and navigate to: http://localhost:3001
  4. The Mental Health Support Hub application will load.

5. Project Structure

  • server.js: Main server file with Express setup and Socket.io integration
  • public/: Directory containing frontend files
    • index.html: Main HTML page
    • styles.css: CSS styles for the application
    • app.js: Client-side JavaScript for functionality
  • package.json: Project configuration and dependencies
  • node_modules/: Installed dependencies (auto-generated)

6. Using the Application

Support Resources

The Resources section displays available mental health support options, including crisis hotlines and therapy links.

Anonymous Chat Support

The Chat section allows users to send anonymous messages in real-time.

  1. Type your message in the input field.
  2. Click "Send" or press Enter.
  3. Messages will appear in the chat area.

Community Forum

The Forum section allows users to post and view community messages.

  1. Enter a title and content for your post.
  2. Click "Post".
  3. Your post will appear at the top of the forum.

7. Development and Customization

Modifying Resources

To add or modify support resources, edit the resources array in server.js.

Styling Changes

Modify public/styles.css to change the appearance of the application.

Adding Features

Extend functionality by editing public/app.js for frontend changes or server.js for backend changes.

8. Troubleshooting

  • Port already in use: Change the PORT in server.js or kill the process using the port.
  • Dependencies not installing: Ensure Node.js and npm are properly installed. Try npm cache clean --force and reinstall.
  • Page not loading: Check that the server is running and the URL is correct.

9. Additional Tools and Libraries Used

  • Express.js: Web framework for Node.js
  • Socket.io: Real-time bidirectional event-based communication
  • Puppeteer: Used for generating PDFs and screenshots (in diagram generation scripts)
  • Docx: Library for creating Word documents
  • Mermaid: For generating diagrams

10. Deployment

For production deployment:

  1. Set the PORT environment variable:
    export PORT=3001
  2. Use a process manager like PM2:
    npm install -g pm2 && pm2 start server.js
  3. Consider using a reverse proxy like Nginx for production.
